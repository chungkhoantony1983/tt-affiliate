## §負担構造の前提 (declared、申込者・利用者 0 円明示)

> 詳細規範: `docs/manual/pj-process-guide.md` §4 §開発インフラ費 + §Running 費用 参照。
>
> - **発注者** = 受注者請求金額 (工数 + DIC 受注者経由 + Running 受注者提供 SaaS 利用料) + 発注者直接契約分を負担
> - **受注者** = 開発を担当 + 本番運用後の SaaS 提供を継続、DIC + Running 受注者提供 SaaS 利用料を発注者に請求
> - **申込者・利用者** (本番運用時の業務利用者、顧客 / エンドユーザー) = **0 円** (自明、明示で誤解防止)
>
> §DIC の負担形態区分 (開発期間中):
> - **受注者経由 (Vendor-Routed)**: 受注者がベンダー契約 + 立替 + 見積に含めて発注者請求
> - **発注者直接契約 (Customer-Direct)**: 発注者が直接ベンダー契約 (見積外、参考値)
>
> §Running の 3 区分 (本番運用後):
> - **§Running 受注者提供 SaaS 利用料 (Service Revenue)**: 受注者 = SaaS 提供企業 の場合、発注者から継続受領する月額利用料 + 従量料金 = **受注者継続収益**、**見積に明示**
> - **§Running 受注者内部原価 (Vendor Cost)**: 受注者が SaaS 提供のために内部支払するベンダー費用 (Azure / Datadog 等)、**見積外** (SaaS マージンで吸収)
> - **§Running 発注者直接契約 (Customer-Direct)**: 発注者が直接ベンダー契約する限定インフラ (専用線 / VPN 等)、**見積外、参考値**
