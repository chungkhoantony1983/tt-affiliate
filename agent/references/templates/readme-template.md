# README テンプレート（ディレクトリ構造 SSoT）

README.md は各ディレクトリの構造定義における SSoT（唯一の正本）として機能する。
全ディレクトリは以下の3レベルのいずれかで README を配置すること。

## レベル判定基準

| レベル | 対象 | 行数目安 |
|:------:|:-----|:-------:|
| Lv.1 | リーフディレクトリ、内容が自明なディレクトリ | 4-8行 |
| Lv.2 | 主要な機能ディレクトリ（複数サブディレクトリを持つ） | 10-30行 |
| Lv.3 | フレームワーク中核ディレクトリ（暗黙知が多い） | 30行超 |

## Lv.1: 最小限（Minimum）

```markdown
# {ディレクトリ名}

{用途を1-2文で記述}

## 内容

- `{ファイル/サブディレクトリ}` — {説明}
```

## Lv.2: 標準（Standard）

```markdown
# {ディレクトリ名}

{用途と設計思想を2-3文で記述}

## ディレクトリ構成

| ディレクトリ / ファイル | 用途 |
|------------------------|------|
| `sub_dir/`             | {説明} |
| `file.ext`             | {説明} |

## 利用方法

{基本的な使い方、コマンド例}

## 関連

- `../{関連パス}` — {関連の説明}
```

## Lv.3: 詳細（Comprehensive）

```markdown
# {ディレクトリ名}

{用途・設計思想・アーキテクチャ上の位置づけ}

## ディレクトリ構成

| ディレクトリ / ファイル | 用途 |
|------------------------|------|
| ...                    | ...  |

## 命名規則

{ファイル命名パターン、プレフィックスルール}

## 実行順序・依存関係

{該当する場合: フック実行順序、スクリプト間依存}

## 変更・追加ルール

{新規ファイル追加時の手順、レビュー要件}

## 関連

{SSoT 相互参照リンク}
```

## 成果物ディレクトリの追加要件

成果物を配置するディレクトリ（`docs/exports/` 等）は、レベルに関わらず以下を記載すること:

- **命名規則**: `{document-name}_{version}_{date}.{ext}`
- **配置先**: 成果物の配置パス
- **エクスポート方式**: `/export` スキル or 手動

これは CLAUDE.md Step 0 b''（成果物配置の確認）で参照される。
