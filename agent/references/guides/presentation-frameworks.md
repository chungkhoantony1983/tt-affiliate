# プレゼンテーション・フレームワーク リファレンス

> プレゼンテーションの作成を個人個人の裁量に委ねていますが、いくつかのフレームワークと原則を提示します。各フレームワークは、プレゼンテーションの目的や対象となる聴衆に応じて選択することが重要です。また、これらのフレームワークは固定的なものではなく、状況に応じて柔軟に組み合わせたり、アレンジしたりすることも可能です。
>
> **なお、プレゼンテーションのフレームワークは、資料作成のみならず、プレゼンの構成そのものを指します。プレゼンの流れも同じ構成を意識してください。**

---

## 1. フレームワーク選択基準

| 目的 | 推奨フレームワーク | 適用レベル |
|------|-------------------|-----------|
| 提案・営業（課題解決型） | PASONA（全体）+ SDS/PREP（各スライド） | 資料全体 |
| 説明・教育 | SDS（全体）+ PREP（各論点） | 資料全体 + 各スライド |
| 理想と現状のギャップ提示 | TAPS（全体）+ SDS（各スライド） | 資料全体 |
| 交渉・合意形成 | DESC（全体）+ PREP（各論点） | 資料全体 |
| ブランディング・認知獲得 | AIDMA（全体）+ SDS（各スライド） | 資料全体 |
| 汎用・社内報告 | SDS（全体）+ PREP（各論点） | 資料全体 + 各スライド |

---

## 2. 各フレームワーク定義

### 2-1. SDS法（Summary → Detail → Summary）

基本的なプレゼン・会話の最小単位。**資料全体にもページ単体にも適用可能。**

| ステップ | 内容 | スライド適用例 |
|---------|------|-------------|
| **Summary** | 冒頭で結論・要点を提示 | スライド冒頭のサブタイトル/リード文 |
| **Detail** | 根拠・詳細を展開 | ボディ部のコンテンツ（カード、チャート、表等） |
| **Summary** | 末尾で結論を再提示・強調 | スライド下部のまとめバナー/結論文 |

**brief での表現:**
```yaml
sds:
  summary: "冒頭結論"
  detail:
    - "詳細項目1"
    - "詳細項目2"
  summary_repeat: "末尾結論（冒頭と同じ趣旨を別表現で）"
```

### 2-2. PREP法（Point → Reason → Example → Point）

よく使われるプレゼンの基本。**資料全体にもページ単体にも適用可能。**

| ステップ | 内容 | スライド適用例 |
|---------|------|-------------|
| **Point** | 主張・結論を明示 | タイトル/サブタイトル |
| **Reason** | 理由・根拠を説明 | 「なぜなら〜」のセクション |
| **Example** | 具体例・データで裏付け | 事例カード、数値、チャート |
| **Point** | 主張を再強調 | まとめバナー/結論文 |

**brief での表現:**
```yaml
prep:
  point: "主張"
  reason: "理由"
  example:
    - heading: "事例1"
      content: "説明"
      data: "数値データ"
  point_repeat: "主張の再強調"
```

### 2-3. PASONAの法則（Problem → Affinity → Solution → Offer → Narrow → Action）

**基本的な資料全体の構成、プレゼン当日の会話の流れはこの法則に則って進められるべき。**

| ステップ | 内容 | 注意点 |
|---------|------|--------|
| **Problem** | 課題を具体的に示す | 数値/データで定量化。未解決リスクも提示 |
| **Affinity** | 共感を示す | 類似事例を共有。感情/懸念の理解を伝える |
| **Solution** | 具体的解決方法を提示 | 実現可能性/効果。複数選択肢のメリデメ |
| **Offer** | 実施手順・投資を提示 | スケジュール、必要リソース。期待成果を数値で |
| **Narrow** | 最適選択肢に焦点を絞る | 判断基準明示。優先順位提示 |
| **Action** | 次のステップを明示 | 責任者/期限設定。フォローアップ方法 |

**brief での表現:**
```yaml
framework:
  overall: PASONA
pasona_mapping:
  problem: slide_01
  affinity: slide_02
  solution: slide_03
  offer: slide_04
  narrow_and_action: slide_05  # Narrow + Action を1枚に統合可
```

### 2-4. TAPS法（To Be → As Is → Problem → Solution）

理想と現状のギャップを明確化し、解決策を提案するフレームワーク。

| ステップ | 内容 |
|---------|------|
| **To Be** | 理想の状態を描写 |
| **As Is** | 現状を客観的に提示 |
| **Problem** | 理想と現状のギャップ（課題）を特定 |
| **Solution** | ギャップを埋める解決策を提案 |

### 2-5. DESC法（Describe → Express → Suggest → Consequence）

状況描写から結果提示まで、納得感を持たせるフレームワーク。

| ステップ | 内容 |
|---------|------|
| **Describe** | 状況を客観的に描写 |
| **Express** | 意見・感情を表現 |
| **Suggest/Specify** | 具体的な提案を行う |
| **Consequence** | 提案の結果（メリット/影響）を提示 |

### 2-6. AIDMAの法則（Attention → Interest → Desire → Memory → Action）

聴衆の心理を段階的に動かすプロセス。ブランディング・認知獲得向き。

| ステップ | 内容 |
|---------|------|
| **Attention** | 注意を引く |
| **Interest** | 興味を持たせる |
| **Desire** | 欲求を喚起する |
| **Memory** | 記憶に残す |
| **Action** | 行動を促す |

## 3. 組み合わせパターン（3層適用）

プレゼンテーションでは、フレームワークを **3層** で適用する:

| 層 | スコープ | 推奨フレームワーク |
|---|---------|------------------|
| **全体構成** | 資料全体のストーリー | PASONA / TAPS / DESC / AIDMA |
| **各スライド** | 1枚のスライド内の構成 | SDS / PREP |
| **各論点** | スライド内の個別論点 | PREP |

### 標準パターン（提案・営業資料）

```
全体: PASONA
  slide_01 (Problem):  SDS — 課題の冒頭結論 → 課題の詳細 → 課題の再提示
  slide_02 (Affinity): SDS — 実績の冒頭結論 → 実績の詳細 → 実績の再提示
    └ 各実績: PREP — 主張 → 理由 → 事例/データ → 再主張
  slide_03 (Solution): SDS — 解決策の冒頭結論 → 解決策の詳細 → 解決策の再提示
  slide_04 (Offer):    PREP — 効果の主張 → 根拠 → 数値事例 → 再主張
  slide_05 (N+A):      SDS — Next Steps の冒頭結論 → ロードマップ → CTA
```

## 4. ブリーフでのフレームワーク指定

ブリーフ（brief.yaml）では以下の構造でフレームワークを指定する（`brief-template.yaml` 参照）:

```yaml
framework:
  overall: PASONA          # 資料全体
  per_slide: SDS           # 各スライドのデフォルト
  per_argument: PREP       # 各論点のデフォルト
```

各スライドでは `sds:` または `prep:` フィールドで具体的な構造を定義する。フレームワークを混在させることも可能（例: slide_02 は PREP、slide_03 は SDS）。
